# Passe QA agentique — 2026-06-14

Première exécution de la boucle `qa-and-refine` (skill `.claude/skills/qa-and-refine`) après la
construction du harnais (#99). Objectif : piloter le DevHost dans des états seedés, juger la qualité
contre la rubrique, corriger dans les jetons existants, réévaluer.

## Méthode

- **Pilotage** : le Playwright MCP de la session est épinglé à Google Chrome (absent de la machine ;
  `npx playwright install chrome` échoue faute de privilèges admin). Repli sans téléchargement :
  capture via le **Chromium bundlé** de Playwright (.NET), utilitaire `QaCapture` (opt-in
  `PISCINE_QA_SHOTS=1`) pointé sur le DevHost lancé par `scripts/devhost-qa.ps1 -Profile mixed`.
- **Matrice capturée** : profil `mixed` × routes `/`, `/cours`, `/rapport`, `/reglages` × thèmes
  clair/sombre × largeurs 1280/420 (16 captures) + journal console par écran.
- Captures dans `%TEMP%/piscine-qa-shots/` (artefacts locaux, non versionnés).

## Constats

### Résolu (P3) — cartes de stats à l'étroit à 420 px
- **Symptôme** : sur le tableau de bord à 420 px, les 5 cartes de stats (Avancement/Fait/En cours/À
  revoir/Restant) restaient sur **5 colonnes fixes** → libellés « En cours » / « À revoir » tronqués/à
  l'étroit.
- **Correctif** (jetons-only) : `BoardOverview.razor.css` — media query `≤ 520 px` →
  `grid-template-columns: 1fr 1fr` (2 colonnes). Desktop inchangé (≥ 520 px garde 5 colonnes).
- **Avant/après** : `mixed-root-light-420.png` (5 colonnes serrées) → `after-root-light-420.png`
  (2 colonnes, libellés lisibles). Build Release 0 warning, 68 tests bUnit verts.

### Signalés (subjectifs / différés — non corrigés d'office)
- **Navbar à 420 px** : enveloppe sur 3 lignes. Acceptable, mais un menu « hamburger » serait plus
  net sur très petit écran — **feature**, pas un correctif jetons → différé.
- **Espace sous le titre `h1`** du tableau de bord (1280) un peu généreux avant la carte « Reprendre »
  — purement esthétique → signalé, non imposé.
- **Numérotation des modules** : saut visible (… 33 → 35 …) dans la liste du board — relève du
  **contenu** (catalogue), hors périmètre UI/CSS.

## Spot-check natif (surfaces propres à Photino, ~5 %) — checklist manuelle

Le DevHost (navigateur) couvre l'UI partagée. À vérifier dans la fenêtre Photino réelle (non
automatisable ici : computer-use non accordé / pas de navigateur MCP) :
- [ ] Barre de titre chromeless : glisser la fenêtre, double-clic = agrandir/restaurer.
- [ ] Boutons fenêtre : réduire / agrandir-restaurer / fermer.
- [ ] Redimensionnement par les bords.
- [ ] Terminal embarqué : ouverture + saisie.

(Le smoke de rendu Photino reste la gate locale : `received:true` prouve l'absence d'écran noir.)
